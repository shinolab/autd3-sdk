稼働中にデバイスとの EtherCAT リンクが健全かどうかを監視したい場合がある.
`Client` はバックグラウンドで状態を監視するタスクを勝手に起動しない.
代わりに, ユーザーが自分のタイミングで駆動する `Checker` を返す.
監視の頻度や, 状態変化への対応をユーザー側で完全に制御できる.

## Checker の取得

通常の [`Client::open`](/autd3-sdk/api/basics/client/) の代わりに `open_with_checker` を使うと, `Client` と `Checker` の組が返る.

## 状態の取得

`Checker::check` は, 呼び出した時点のリンク状態 `LinkStatus` を返す.
`LinkStatus` は以下を持つ.

- `devices`: デバイスごとの状態 (`DeviceState`). 正常なら `OP`, 切断されると `LOST` など.
- `all_op`: 全デバイスが `OP` 状態か.
- `any_lost`: いずれかのデバイスが `LOST` 状態か.
- `recoveries`: 切断から復帰した累計回数.
